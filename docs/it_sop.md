# 全球 IT 維運標準作業程序與合規指南 (IT SOP)

**文件版本：** v2.4 (2026 最新修訂版)
**適用範圍：** 全球營運據點之全體員工與 IT 維運團隊
**核心精神：** 零信任架構 (Zero Trust)、最小權限原則 (PoLP)、雙軌會簽防護。

---

## 1. 總則
本文件旨在規範企業內部 IT 資源之申請、變更、故障排除與資產管理流程。所有涉及權限異動、軟體安裝與基礎設施變更之需求，均須透過 IT 維運平台開立工單 (Ticketing)，並依據風險分級 (Tier Levels) 完成相應之審批流程。未經授權之操作，將視為重大資安違規。

---

## 2. 軟體與系統權限風險分級 (Tier Levels)

為確保企業資安與預算控管，所有 IT 申請項目嚴格劃分為四個風險等級。申請人提出需求時，系統將自動依據「影響範圍」與「預算金額」判斷其 Tier 等級。

### Tier 1：低風險 / 自助服務 (Low Risk)
* **定義：** 不涉及機密資料存取、無額外預算或授權費用、且員工可自助完成之項目。
* **常見項目：** AD 帳號密碼重設、清除系統快取、查詢 IT 規範。
* **簽核要求：** **免簽核**。系統可自動核准並執行。

### Tier 2：中風險 / 部門級別 (Medium Risk)
* **定義：** 涉及部門內部資源存取，或預算低於新台幣 10 萬元之標準設備/軟體採購。
* **常見項目：** 申請安裝標準辦公軟體、部門共用資料夾權限、單一 IT 設備採購。
* **簽核要求：** 需過 2 關。User 端直屬經理 ➡️ IT 維運經理。

### Tier 3：高風險 / 處級別 (High Risk)
* **定義：** 涉及核心資料庫存取、本機管理者權限，或預算介於新台幣 10 萬至 500 萬之間之專案與擴充。
* **常見項目：** 專業 IDE/資料庫軟體授權、跨部門系統連線開放、既有機房設備擴充。
* **簽核要求：** 需過 4 關。Tier 2 流程 ➡️ User 處長 ➡️ IT 處長。

### Tier 4：極高風險 / 企業級別 (Extreme Risk)
* **定義：** 涉及影響全公司營運之基礎設施變更、具備毀滅性之實體操作，或預算高於新台幣 500 萬之大型建置案。
* **常見項目：** 千萬級大型機房新建、核心生產環境 (Production) 架構變更、遠端抹除遺失設備。
* **簽核要求：** 需過 7 關。Tier 3 流程 ➡️ 副總 (VP) ➡️ 總經理 (GM) ➡️ 董事長 (Chairman)。

## 3. 雙軌會簽矩陣與審批邏輯 (Approval Matrix)

企業採用「業務端 (Business)」與「技術端 (IT)」雙軌會簽機制，以確保預算與資安雙重合規。

1.  **業務端 (User 端) 審批原則：** 負責確認該申請是否符合部門業務需求，以及是否願意承擔該項目產生之部門預算與費用。
2.  **技術端 (IT 端) 審批原則：** 負責評估該申請是否符合企業資安規範、現有架構是否支援、以及確認可用之軟體授權 (License) 數量。
3.  **審批順序：** 必須由「業務端」先行核准後，方可送交「技術端」會簽。若業務端退回，則 IT 端無需處理。

---

## 4. 退簽與補件規範 (Rejection & Revision)

在簽核流程中，任何層級之主管皆具備退簽或要求補件之權力。

* **退回前一關：** 若主管認為前一層級之審核不夠嚴謹 (如 IT 處長認為 IT 經理的資安評估不完整)，可將工單退回給前一關主管重新審查。
* **退回申請者：** 若申請事由交代不清、缺乏必要之附件 (如架構圖、報價單)，主管可直接將工單退回給申請人。申請人需於 3 個工作日內補充資訊後重新送單，逾期系統將自動結案 (Closed_Incomplete)。
* **專家會簽 (加簽)：** 主管於審批時，若遇跨領域之專業技術問題，得暫停審批，並發出邀請將特定領域專家 (如 資安長 CISO、Data Owner) 加入該關卡進行平行會簽。

## 5. 設備遺失與緊急資安事件響應

當員工回報企業配發之筆記型電腦或行動裝置遺失時，處理流程如下：
1.  員工必須第一時間透過 Agent 回報設備遺失。
2.  系統將自動建立「緊急資安事件 (Incident)」工單，並判定為 Tier 4 極高風險操作。
3.  為防止商業機密外洩，此類特殊 Tier 4 案件將啟動「緊急授權通道 (Emergency Access)」，僅需 IT 處長與資安長 (CISO) 兩人平行會簽同意後，系統即可強制
