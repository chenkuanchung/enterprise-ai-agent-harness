# GlobalTech IT 維運標準作業程序 (SOP) v2026.1

## 1. 帳號與權限管理規範 (IAM)
- **帳號解鎖**：遇到員工帳號被鎖定 (Locked)，維運人員可直接執行解鎖。但解鎖完成後，【必須】口頭或傳訊提醒使用者：「請務必在 24 小時內登入系統重設密碼」。
- **超級管理員權限**：嚴禁透過系統自動授予 SuperAdmin 權限。任何 SuperAdmin 申請皆須由「資安部總監 (CISO)」親筆簽名核准。

## 2. 設備與容量管理規範 (CMDB)
- **容量不足定義**：設備剩餘硬碟空間小於 10,000 MB (10GB) 時，判定為容量不足。
- **清除快取SOP**：若發生容量不足，可先執行「清除系統快取」。
- **硬體升級SOP**：如果執行清除快取後，剩餘空間【依然小於 10GB】，請勿再重複清除。必須立刻中止操作，並回報給「硬體採購部 (Hardware Procurement Dept)」進行 SSD 擴充評估。
- **遠端抹除設備 (Remote Wipe)**：這是極高風險操作。【必須】將工單狀態轉為 Pending_Approval，並強制觸發主管審批。取得核准後方可呼叫抹除工具。

## 3. 軟體安裝與授權規範 (Software Provisioning)
企業內部軟體依據資安風險與授權成本分為三個等級 (Tiers)。當使用者申請安裝軟體時，Agent 必須嚴格遵循以下流程：

- **Tier 1 (一般開源軟體)**：如 7-Zip, VS Code, Google Chrome。
  - **處置流程**：無需開立工單，無需主管審批。Agent 可直接回覆安裝指南或核准請求。

- **Tier 2 (受控開發軟體)**：如 Docker Desktop, Postman, Wireshark。
  - **處置流程**：
    1. 【必須】呼叫 create_ticket 工具開立 ITSM 追蹤工單。
    2. 【必須】在工單建立後，先將工單資訊與目的提供給申請者檢查，詢問「請問工單內容是否正確？」。
    3. 確認無誤後，即可同意安裝 (此等級無需主管簽核)。

- **Tier 4 (高風險與付費授權軟體)**：如 VMware, Navicat, 任何 VPN Client。
  - **處置流程**：
    1. 【必須】呼叫 create_ticket 工具開立工單。
    2. 【必須】向申請者確認工單內容無誤。
    3. 【必須】確認無誤後，呼叫 update_ticket_status 將狀態改為 `Pending_Approval`。
    4. **【強制中斷】**：操作至此必須凍結系統，強制觸發主管人工簽核 (HITL)。未取得主管核准前，嚴禁回覆安裝成功。
